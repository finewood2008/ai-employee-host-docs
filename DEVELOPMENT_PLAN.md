# 产品开发计划

## 阶段 0：文档与方向确认

状态：已完成初版。

产出：

- 商业计划书
- Agent Builder 产品设计
- GUI 模板库
- 高保真 HTML 原型
- CTO 交接说明

## 阶段 1：前端高保真 Demo 工程化

目标：把 HTML 原型拆成可维护的 React 工程。

任务：

- 初始化 React + TypeScript 项目
- 建立 AppShell
- 建立数字员工模板导航
- 实现 KPIGrid
- 实现三栏工作台布局
- 实现资料处理型模板
- 实现任务跟进型模板
- 实现汇报分析型模板
- 使用 mock 数据驱动页面

验收标准：

- 可在浏览器或 Electron 中打开
- 可以切换 3 个核心员工模板
- 页面数据来自配置对象，而不是纯写死 JSX

## 阶段 2：本地数据与文件能力

目标：让 Demo 具备本地资料处理基础能力。

任务：

- 引入 SQLite
- 设计客户表、文件表、任务表、日志表
- 支持本地文件导入
- 支持文件元数据记录
- 支持 mock 文件分类
- 支持客户匹配建议
- 支持人工确认归档

验收标准：

- 用户可以导入一批 mock 文件
- 系统可以展示识别结果
- 系统可以生成归档建议
- 用户可以确认或标记异常
- 所有动作写入日志

## 阶段 3：财税 MVP 闭环

目标：完成资料整理员 → 客户催收员 → 老板日报员的闭环。

任务：

- 资料整理员生成缺失资料清单
- 客户催收员读取缺失资料清单
- 客户催收员生成催收话术
- 人工确认话术
- 模拟企微/飞书发送
- 记录客户回复状态
- 老板日报员汇总进度

验收标准：

- 能演示完整链路
- 每一步都有可视化页面
- 每一步都有日志
- 每一步都能解释 AI 判断依据

## 阶段 4：Agent Builder MVP

目标：通过自然语言创建一个客户催收员。

任务：

- 设计 Builder 对话页
- 设计岗位蓝图数据结构
- 通过对话抽取岗位信息
- 生成 EmployeeViewConfig
- 生成测试任务
- 生成上线清单

验收标准：

- 输入一句岗位需求
- 系统生成客户催收员蓝图
- 系统生成 GUI 配置
- 系统生成测试任务
- 用户确认后进入员工工作台

## 阶段 5：真实渠道与模型接入

目标：从 mock 进入真实可用。

任务：

- 接入模型网关
- 接入企微/飞书测试通道
- 接入真实文件 OCR / 文档解析
- 支持权限与审计
- 支持错误重试

验收标准：

- 可在测试环境真实发送消息
- 可处理真实样例文件
- 可记录完整审计日志
- 可配置人工确认节点

## 当前优先级

最高优先级：

1. 前端 Demo 工程化
2. EmployeeViewConfig 配置模型
3. 资料处理型模板
4. 任务跟进型模板
5. 汇报分析型模板
